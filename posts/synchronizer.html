Waiting on arbitrary condition in Java

I was playing a little with the synchronization primitives in Java lately. Every Java programmer knows basic mechanism: you wait on some object and notify one or all threads from some other thread. Every Java object has monitor object associated with it on which you synchronize. But here is cool problem: what if you wanted to wait on some logical condition?. I was thinking about it and I think I solved it. What follows is nifty little tool with code with explanations.

The problem

You want to put the thread to sleep and awaken it only if some logical condition is true. So basically you need to invoke wait() on some object and call notify() from some other thread. But this is not close yet. The problem is how you enforce the condition.

What we can do is to associate lock object with Predicate<E>. First create the object that will serve as a lock. Then associate it with the predicate. And then call wait on this object. Some time later other thread will get that pair and run time predicate with some value. If predicate evaluates to true then it will call notify. Because we have one thread per lock this particular thread waiting will be woken up.

Here how it looks in the code...

The code


 public class Synchronizer<T> {
    record LockEntry<E>(Predicate<E> condition, Object lock) {
    }

    private final LinkedList<LockEntry<T>> locks = new LinkedList<>();

    public void waitOnCondition(Predicate<T> condition, long timeout) {
        var lock = new Object() {
        };

        locks.offerLast(new LockEntry<T>(condition, lock));

        synchronized (lock) {
            try {
                lock.wait(timeout);
            } catch (InterruptedException e) {
                throw new RuntimeException("InterruptedExeption", e);
            }
        }
    }

    public void wakeWith(T value) {
        Iterator<LockEntry<T>> it = locks.iterator();
        while (it.hasNext()) {
            var entry = it.next();

            if (entry.condition.test(value)) {
                synchronized (entry.lock){
                    entry.lock.notify();
                }
                it.remove();
            }
        }
    }
}

First off we are using new records (Java 16) to have Predicate-lock mapping. We put it in the list. The condition has some closure variables that it checks. It can be as simple as checking the id or as complex as waking only prime ids (for ex.).

The other thread uses wakeWith(T value) feed the predicate evaluation. It iterates the mentioned list and if predicate==true then notifies the thread and removes the entry.

Quick test

Let's see if it works. To test medium complexity use case, lets spawn 4 threads and wake only the half with the same modulo number (odd-even).


public class SynchronizerTest {

    @Test
    void test_only_even_numbers_pass() throws InterruptedException {
        int[] arr = new int[]{-1, -1, -1, -1};
        var synchronizer = new Synchronizer<Integer>();

        for(int i=0;i<4;i++){
            var k = i;
            new Thread(() -> {
                synchronizer.waitOnCondition((Integer j) -> j % 2 == k % 2, 10_000);
                arr[k] = k;
            }).start();
        }

        Thread.sleep(400);

        synchronizer.wakeWith(1);

        Assertions.assertArrayEquals(new int[]{-1, 1, -1, 3}, arr);
    }
}

We wake with 1 so only odd threads are woken up.
