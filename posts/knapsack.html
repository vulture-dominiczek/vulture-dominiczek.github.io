Knapsack problem and weighted median

Knapsack problem is a leading example for using the dynamic programming. There are many variants of which two are most prominent fractional and discrete. I will shortly discuss both, but concentrate on the fractional one because the link between weighted median and it's solution is really interesting.

Dynamic programming and discrete case

The problem is like this: 


Given 1..n items each with cost c[i] and weight w[i], fill in the backpack with weight W so that
the value of the packed items is maximized.

The naive way seems to be using greedy algorithm as follows: sort the items so they are descending by cost, fill in the backpack taking items in order until we reach capacity. But it's wrong. It's because the last item can not fit in the backpack in whole leaving empty space in the backpack. And this space lowers the overall efficiency of the solution.

This problem is solvable with dynamic programming. It is because it has optimal subproblems property. For the optimal solution W2 which is subsolution W1 plus optimal item w[k] in this step the W1 subsolution is also optimal. Given this we can implement O(nW) solution with dynamic programming, computing:


S[W] = max [S[W-w[i]] + weight[i]]

Greedy approach and fractional case

This is the main part of the article. In this case we can cut the items into pieces to fill in the backpack at full. This time we can solve it greedily because at all times we have backback filled to it's full. But more suitable problem statement would be as follows.


Given 1..n liquids with density c[i] and velocity v[i] fill in the bucket of velocity V so that the weight
of the mixed liquids is maximized.

To obvious approach is to sort the liquids by density and take them one by one until V is reached. But this solution takes O(nlogn) time because of sorting. Can we do it faster? Yes!

With the help of Quick Select we can do this in O(n) time. The algorithm is as follows:


S = (sum of all weights) - V


SELECT(left, right, S)
    pivot <- choose random index in range [left, right]
    i <- PARTITION(left, right, pivot)
    L <- sum of w[i] in partition left to the i
    R <- sum of w[i] in partition right to the i

    if L > S:
        SELECT(left, i, S)
    else if R > S:
        SELECT(i+1, right, S-L-w[i])

PARTITION(left, right, pivot)
    procedure moves items with smaller c[i] than c[pivot] to the left,
    and greater to the right

And the result is in right partition. As it occurs this is just the special case for finding weighted k-th statistic. If you think of weighted k-th statistic problem it reduces to the normal statistic with duplicates. For w[i] we duplicate element c[i] w[i] times. The algorithm is exactly the same either greedy of quick select.

Conclusion

With the use of quick select we solved weighted median problem in linear time.
