Finding overlapping rectangles

A problem from Cormen:

VLSI databases commonly represent an integrated circuit as a list of rectangles. Assume that each rectangle is rectilinearly oriented (sides parallel to the xxx- and yyy-axes), so that we represent a rectangle by its minimum and maximum xxx and yyy-coordinates. Give an O(nlgn)-time algorithm to decide whether or not a set of n rectangles so represented contains two rectangles that overlap. Your algorithm need not report all intersecting pairs, but it must report that an overlap exists if one rectangle entirely covers another, even if the boundary lines do not intersect. (Hint: Move a "sweep" line across the set of rectangles.)

This is doable using interval tree. Let's see how to do it...

Intersection of rectangles

When do the rectangles intersect?

According to trichotomy of intervals, they can be in any of the three states:

  • separate, first interval first, second interval second
  • separate, second interval first, first interval second
  • intersecting

Then we would state that the intervals intersect when

left1 <= right2 && right1 <= left2.

The same goes for the rectangles but we use a conjunction of the up-down and left-right bounds. For rectangle intersection is when:

left1 <= right2 && right1 <= left2

and

up1 <= down2 && up2 <= down1

conditions are met.

Interval tree

This is augumented balanced binary tree (red black tree for example) that allows to check intersection of given interval in O(logN) time. The idea is to have intervals sorted by left end this being the key of the node while also holding maximum x of right end that the subtree holds. Thanks to this while searching we can rule out either left or right subtree and this gives us the O(logN) time.

Algorithm

  • We assume that we have collection of rectangles R, that each holds four vertices {r1, r2, r3, r4}.
  • We sort R by left[R].
  • We build interval tree of R by rectangles intervals down[R]-up[R].
  • We iterate sorted list of rectangles if the next rectangle intersects on X then we look up the associated vertical interval in interval tree. If there are more than 1 result, then we found intersecting rectangles.
  • Iteration - O(N) and search in interval tree O(logN).
