Problem and solution

This is a problem I chose. We have to find longest substring which doesn't have repeating characters in it. The solution is very simple, with hashset in place it is O(n).

So the idea is to scan string from left to right with i index. While we are progressing to next characters we are adding them to hashset which holds unique elements. But if we find that new char is already in set we start removing characters from the back of our substring. The j index is pointing to the first not removed yet char in our substring. So our substring is spanning from j to i and it's length is i-j+1 with j stricly less than i. We keep progressing j and removing chars from the set until set no longer contains char at i (or j hits i). This is how it would look like on example



We keep adding chars to the set...

1. i=0, j=0, string="abcabc", set={}

2. i=1, j=0, string="abcabc", set={'a'}

3. i=2, j=0, string="abcabc", set={'a', 'b'}

4. i=3, j=0, string="abcabc", set={'a', 'b', 'c'}

At string[3] we have 'a' again so set.contains('a') is true. 
We keep removing chars from set and progressing j

4.1. j=3, j=1, string="abcabc", set={'b', 'c'}
and finally add 'a'

5. i=4, j=1, string="abcabc", set={'a', 'b', 'c'}

5.1 i=4, j=2, string="abcabc", set={'a', 'c'}

6. i=5, j=2, string="abcabc", set={'a', 'b', 'c'}

6.1 i=5, j=3, string="abcabc", set={'a', 'b'}

7. i=6, j=3, string="abcabc", set={'a', 'b', 'c'}

Rust solution

And now lets see how it is done in Rust...


use std::collections::HashSet;
use std::cmp;

impl Solution {
    pub fn length_of_longest_substring(s: String) -> i32 {
        let mut letters = HashSet::new();
        let chars:Vec<char> = s.chars().collect();
        let mut j = 0;
        let mut i = 0;
        let mut maxLen = 0;
        
        while i < chars.len() {
            let c = chars[i];
            if !letters.contains(&c) {
                letters.insert(c);
            }
            else {
                while j < i && letters.contains(&c){
                    let cj = chars[j];
                    letters.remove(&cj);
                    j += 1;
                }
                letters.insert(c);
            }
            maxLen = cmp::max(maxLen, i-j+1);
            i += 1;
        }
        
        return maxLen as i32;
    }
}

There are couple of elements in play let's go through them one by one.

Mutablility in Rust (as in other languages) there are two ways to allocate objects: on heap or stack. To allocate objects on stack we must know it's exact size. Further to allow modification we have to annotate variable with mut if we want to reassigning it. That's why we have i and j mutable as well as maxLen and letters.

Sizedness check out this great post. Generally if types size can be measured at compile time it can be passed as value and reside on stack. All primitive types and structs and enums (recursively) are sized types.

Copy as you probably noticed we are assingning chars to c and cj. In fact we are creating copies because primitive types implement Copy trait and are sized. What this means that on variable moving we are creating bitwise copy of previouse element.

Borrowing most of the passing arguments to functions in Rust happens by borrowing. Borrowing is opposite to variable moving because we don't take ownership of the variable inside function. However it has an limitation that we can't modify object with this reference (but we can copy it).

Casting Rust doesn't support widening conversions and we have to explicitly cast types. This is done with operator as
