Rust by example: simple sniffer

Some time ago I was learning about linux'es network stack and started by looking at the traffic in Wireshark. Meanwhile I was learning Rust so decided to write myself a simple PCAP-using tool. I found it easier than I thought and result is more like a script than package. Here I will be sharing my experiences and what I learned.

PCAP

PCAP is an API for programmers to write network monitoring tools. It originated in Berkeley Laboratory, and itself was attempt to unificate code of various system distributions. The attempt succeeded and currently available on MacOS, Windows and Linux. The most widely known tool using it is tcpdump which is further used by graphical interfaces like Wireshark.

What you get from this C library is raw layer 2 data. Most widely it is Ethernet v2 but can be any Link Layer protocol and there are many of them. PCAP talks directly to the network card and mirrors the traffic independently of OS network stack. You can even inject some frame to the stack. However, this is a raw data which needs to be parsed, and this what we will be doing.

Networking theory

Quick theory recap: networking traffic is organized into layers. Lowest layer is Phisical Layer and this is 1's and 0's flowing throught the wire or over the radio antenna. Next there is Link Layer and it is a language of various networking devices like bridges, switches and network cards. Then there is logical IP that with the use of routers knows where to transport packet anywhere in the world. But IP itself doesn't handle fragmentation so nexl level layers TCP or UDP allow to send much larger segments.

Each time the frame is sent it goes from the upper layer to the lower layer. The binary data is encapsulated or wrapped in lower layer's frame. When it is beeing received the reverse process happens - it is decapsulated.

Rust interlude

Rust is multi-paradigm language and there are many ways to describe same thing. Robert C. Martin in his Clean Code series observed that Object Oriented and Functional Programming are complementary. In OO you have dynamic dispatch and you hide differnt object types behind common interface. By doing so it is easy to extend structure but hard to extend behaviuour. On FP side of things opposite is true. You just have pure functions operating on structures. Each time you want to add behaviour you just write another function but when you change structure you have to reimplement all functions.

It appears to me that Rust leans more towards Functinal Programming. True there are dynamic trait objects but their use is cumbersome and "all around". The rich enums and pattern matching seems to be cleaner way to implement stuff and writing in this style was pure pleasure for me. However I'm still learning the language so this statement may be premature.

While writing my little script I had to look up the frame formats in various sites and RFC's and I'm putting this data here for your convenience. For clarity I ommitted some values but you can find them in code.

Implementing

I started by defining structures. Thanks to using enum we can use rich pattern matching which is easy to write. We wrap raw data in enum value giving it meaning. This is a newtype pattern which is called so by Rust community after some construct in Haskell. In short you wrap existing type in enum or struct. Then you hide olders type implementation details behind your new type. Because you have new type you can implement traits for it or just write plain functions.


enum Record<'a> {
    Ethernet2Frame(&'a [u8]),
    Ip4Packet(&'a [u8]),
    Ip6Packet(&'a [u8]),
    TcpSegment(&'a [u8]),
    UdpPacket(&'a [u8]),
}

Next for source address, destination address, payload I wrote parsing in separate methods. Nice thing in Rust is functional api for operating on collections. Here on print_dot example I get the iterator, map values to strings, join them together with dot. The result is option, but I know it always is Some so I can safely unwrap.

Then in source_address I used pattern matching to tie parsing with frame type. In Rust you can destructure the type to get it's internals so I use that to get b: &[u8] inside enums variant.


fn print_dot(data: &[u8]) -> String {
    data.to_vec()
        .iter()
        .map(|x| format!("{}", x))
        .reduce(|a, b| format!("{}.{}", a, b))
        .unwrap()
}

fn print_hex(data: &[u8]) -> String {
    data.to_vec()
        .iter()
        .map(|x| hex::encode([*x]))
        .reduce(|a, b| format!("{}:{}", a, b))
        .unwrap()
}

fn source_address(rec: &Record) -> String {
    match rec {
        Record::Ethernet2Frame(b) => print_hex(&b[..6]),
        Record::Ip4Packet(b) => print_dot(&b[12..16]),
        Record::Ip6Packet(b) => print_hex(&b[8..24]),
        Record::TcpSegment(b) => {
            let mut addr = (b[0] as u16) << 8;
            addr += b[1] as u16;
            format!("{}", addr)
        }
        Record::UdpPacket(b) => {
            let mut addr = (b[0] as u16) << 8;
            addr += b[1] as u16;
            format!("{}", addr)
        }
    }
}

The most interesting part is decapsulation. What is interesting here that we jump from one enum's variant to another or return None if this is some case we didn't consider yet. And so we can decapsulate Etherenet2 to Ipv4 and Ipv6. Then we can decapsulate either Ipv4 or Ipv4 to Tcp or Udp. Note that the capturing whole case is done with @ in Rust.


fn decapsulate<'a>(rec: &'a Record) -> Option<Record<'a>> {
    match rec {
        eth @ Record::Ethernet2Frame(b)=> {
            let ethertype = &b[12..14];

            if hex::encode(ethertype) == "0800" {
                Some(Record::Ip4Packet(payload(eth)))
            }
            else if hex::encode(ethertype) == "86dd" {
                Some(Record::Ip6Packet(payload(eth)))
            }
            else {
                None
            }
        },
        ip4 @ Record::Ip4Packet(b) => {
            if b[9] == 17 {
                Some(Record::UdpPacket(payload(ip4)))
            }
            else if b[9] == 6 {
                Some(Record::TcpSegment(payload(ip4)))
            }
            else {
                None
            }
        },
        ip6 @ Record::Ip6Packet(b) => {
            if b[6] == 17 {
                Some(Record::UdpPacket(payload(ip6)))
            }
            else if b[6] == 6 {
                Some(Record::TcpSegment(payload(ip6)))
            }
            else {
                None
            }
        },
        _ => None
    }
}

The last part is main method, I quickly implemented argument parsing with Clap and filtering of the payloads is done by JSON path which is done by jsonpath-rust and serde_json. The original concept was that if the jsonpath query matches then let's print the result in the commandline.


fn main() {
    let device = Device::lookup().unwrap();

    let mut pcap = Capture::from_device(device)
        .unwrap()
        .promisc(true)
        .immediate_mode(true)
        .snaplen(5000)
        .open()
        .unwrap();

    let args = Args::parse();


    loop {
        if let Ok(packet) = pcap.next() {
            let eth = Record::Ethernet2Frame(packet.data);

            let mut root = serde_json::Map::new();

            let ethernet = serde_json::json!({
                "source": source_address(&eth),
                "destination": destination_address(&eth)
            });

            root.insert(name(&eth), ethernet);

           
            if let Some(ip) = decapsulate(&eth) {
                let ip_packet = serde_json::json!({
                    "source": source_address(&ip),
                    "destination": destination_address(&ip),
                });

                root.insert(name(&ip), ip_packet);

                if let Some(l4) = decapsulate(&ip){
                    let l4_packet = serde_json::json!({
                        "source": source_address(&l4),
                        "destination": destination_address(&l4),
                    });

                    root.insert(name(&l4), l4_packet);
                }
            }


            let mut s = String::from("");
            for b in packet.data {
                let c = char::from(b.clone());
                if c.is_alphanumeric() {
                    s.push(c);
                } else {
                    s.push_str(".");
                }
            }
            root.insert(String::from("data"), serde_json::Value::String(s));

            let obj = serde_json::Value::Object(root);

            if let Ok(res) =  &obj.clone().path("$..book[?(@.author size 10)].title") {
                println!(
                    "{}",
                    serde_json::to_string_pretty(&obj).unwrap()
                )
            }
           
        }
    }
}
